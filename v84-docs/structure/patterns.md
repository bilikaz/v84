# Patterns

Verified working code snippets for installed packages. Each file shows exactly how we use that package in this project.

## How to maintain

- When you install a package and figure out the correct usage, add a pattern file
- When a package updates and breaks things, update the pattern file with what works now
- If you get errors you can't solve, paste the error into your AI assistant and ask it to update the pattern file
- Only add files for packages you actually use — don't pre-create empty files
